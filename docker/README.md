# AlgoWebApp (Docker Compose)
Взаимодействие между сервисами изолировано внутри внутренней сети Docker.

1. Gateway (Nginx)

Входная точка приложения (Reverse Proxy).
**Порт:** `8080` (внешний) -> `80` (внутренний).
**Функции:**
    - Проксирование запросов на фронтенд (`/`).
    - Проксирование API-запросов на сервис авторизации (`/api/auth/` -> `/auth/`).
    - Сжатие ответов (Gzip).
    - Обработка заголовков для передачи реального IP пользователя.


2. Frontend
Клиентское приложение.
- **Сборка:** Многоэтапный Dockerfile (Node.js 22 для билда, Nginx для раздачи статики).
- **Порт:** `3000` (внешний).
- **Зависимости:** Ожидает запуск `auth-service`.

3. Auth Service
Микросервис авторизации на базе Java.
- **Порт:** `8081`.
- **Конфигурация:** Использует `.env` файл и переменные окружения для настройки JWT, времени жизни токенов и параметров БД.
- **Зависимости:** Не запустится, пока `auth-db-postgres` не пройдет проверку `healthcheck` (состояние `healthy`).

3. Auth DB (PostgreSQL 16)
База данных для сервиса авторизации.
- **Образ:** `postgres:16-alpine`.
- **Данные:** Сохраняются в именованном volume `auth-db-postgres`.
- **Healthcheck:** Проверяет готовность БД через `pg_isready`.

---

Сетевая структура
Все сервисы объединены в bridge-сеть `algo-network`. Это позволяет обращаться к сервисам по их именам (например, `http://auth-service:8081`).

**Запуск всех сервисов в фоне:**
```
docker-compose up -d
```

**Остановка и удаление контейнеров с томами:**

```
docker-compose down -v
```
**Просмотр логов конкретного сервиса:**

```
docker-compose logs -f auth-service
```
